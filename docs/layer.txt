# 계층

* Web Layer
>> 컨트롤러(+ 어드바이스), 필터, 인터셉터 및 뷰 템플릿 영역
>> 외부 요청과 응답에 대한 전반적인 영역

* Service Layer
>> 컨트롤러와 Dao 사이의 영역 (일반적 의미)
>> @Transactional 이 사용되어야 하는 영역
>> 서비스 계층은 비즈니스 로직을 처리하는 역할이 아니라 트랜잭션과 도메인 간 순서 보장 역할
   (서비스 계층에서 비즈니스 로직을 처리하는 방식을 '트랜잭션 스크립트'라 하며 이 실습에서 사용하지 않는다.)

* Repository Layer
>> 데이터 저장소에 접근하는 영역

* Dto
>> 계층 간 데이터 교환을 위한 객체

* Domain Model
>> 개발 대상을 모든 사람이 동일한 관점에서 이해할 수 있도록 단순화시킨 것
   (택시 애플리케이션을 가정할 때, 도메인 모델은 배차, 요금, 탑승 등을 의미한다.)
>> @Entity 가 사용된다면 도메인 모델이다.
>> VO(값 객체)와 같이 데이터베이스의 테이블과 무관하더라도 도메인 모델이 될 수 있다.
>> 비즈니스 로직이 수행된다.
